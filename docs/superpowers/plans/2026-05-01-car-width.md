# Car Width Attribute Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a per-car `width` attribute (meters, body width perpendicular to the constraint line). Today the renderer hardcodes `CAR_WIDTH = 2.5`; this plan makes width editable in the train editor, persisted in the car-definition file format, carried on every runtime `Car`, serialized to scene files, and consumed by the train renderer.

**Architecture:** Width is added at every layer that already knows about `edgeToBogie`/`bogieToEdge` — the same plumbing path. A new numeric input in the train-editor toolbar drives `BogieEditorEngine.exportCarDefinition`, which adds `width` to the returned shape. `CarDefinitionData`, `CarTemplate`, the runtime `Car` class, and `SerializedCar` all gain `width`. The bogie-editor render system gets a body-outline rectangle so the user can see what they're editing. The train renderer reads `car.width` instead of the constant. Default is **2.5 m** to match today's rendered behavior; all loaders fall back to 2.5 for files saved before this feature, so old saves render identically.

**Tech Stack:** TypeScript, Bun test runner (`import from 'bun:test'`), React + Tailwind (toolbar UI), PIXI.js (bogie editor rendering).

**Spec reference:** No formal spec — scope is taken from the conversation that produced this plan.

**Conventions:**

- Width is the **full** body width in meters, not half-width. (The existing `DEFAULT_CAR_HALF_WIDTH = 1.5` constant in platform-offset.ts uses half-width and is unrelated to this work.)
- Default value everywhere: `2.5`.
- When deserializing legacy data without `width`, fall back to `2.5` silently — no warning, no migration.

**Out of scope (do NOT do these in this plan):**

- Wiring `car.width` into `computePlatformOffset` placement. Platforms are placed without knowing which cars will visit; that needs separate design.
- Drag handles for resizing width in the bogie editor. Numeric input only.
- Reconciling `DEFAULT_CAR_HALF_WIDTH` (3 m default in platform-offset) vs. renderer default (2.5 m). Different concerns, intentionally left as-is.

```typescript
    exportCarDefinition(
        edgeToBogie: number = 2.5,
        bogieToEdge: number = 2.5,
        width: number = 2.5
    ): {
        bogieOffsets: number[];
        edgeToBogie: number;
        bogieToEdge: number;
        width: number;
    } | null {
        if (this._bogies.length < 2) return null;

        const dir = this._lineDirection;
        const origin = this._lineOrigin;
        const projections = this._bogies.map(bogie => {
            const dx = bogie.x - origin.x;
            const dy = bogie.y - origin.y;
            return dx * dir.x + dy * dir.y;
        });

        const sorted = [...projections].sort((a, b) => a - b);

        const bogieOffsets: number[] = [];
        for (let i = 1; i < sorted.length; i++) {
            bogieOffsets.push(sorted[i] - sorted[i - 1]);
        }

        return { bogieOffsets, edgeToBogie, bogieToEdge, width };
    }
```

```typescript
import type { Point } from '@ue-too/math';

import type { CarType } from './cars';

export type CarTemplate = {
    id: string;
    bogieOffsets: number[];
    edgeToBogie: number;
    bogieToEdge: number;
    /** Body width in meters. */
    width: number;
    type?: CarType;
    image?: {
        src: string;
        position: Point;
        width: number;
        height: number;
    };
};

let _templateIdCounter = 0;

export function generateTemplateId(): string {
    return `tpl-${_templateIdCounter++}`;
}

export function validateCarDefinition(
    data: unknown
): { valid: true } | { valid: false; error: string } {
    if (typeof data !== 'object' || data === null) {
        return { valid: false, error: 'Not an object' };
    }
    const obj = data as Record<string, unknown>;
    if (!Array.isArray(obj.bogieOffsets) || obj.bogieOffsets.length === 0) {
        return { valid: false, error: 'Missing or empty bogieOffsets array' };
    }
    if (
        !obj.bogieOffsets.every((v: unknown) => typeof v === 'number' && v > 0)
    ) {
        return { valid: false, error: 'bogieOffsets must be positive numbers' };
    }
    if (obj.width !== undefined && typeof obj.width !== 'number') {
        return { valid: false, error: 'width must be a number when provided' };
    }
    if (obj.width !== undefined && (obj.width as number) <= 0) {
        return { valid: false, error: 'width must be positive' };
    }
    return { valid: true };
}
```

- [ ] **Step 1: Expose width on the engine**

In `src/train-editor/bogie-editor-engine.ts`, add (near other state):

```typescript
    private _width: number = 2.5;
    private _widthListeners: Array<(width: number) => void> = [];

    get width(): number {
        return this._width;
    }

    setWidth(value: number): void {
        if (value === this._width) return;
        this._width = value;
        for (const cb of this._widthListeners) cb(value);
    }

    onWidthChanged(cb: (width: number) => void): () => void {
        this._widthListeners.push(cb);
        return () => {
            const i = this._widthListeners.indexOf(cb);
            if (i >= 0) this._widthListeners.splice(i, 1);
        };
    }
```

```typescript
    ): { ... } | null { ... }
```

- [ ] **Step 2: Draw the body rectangle in the render system**

In `src/train-editor/bogie-editor-render-system.ts`, add a `Graphics` instance for the body, populate it on `_syncAll`, and update on `onWidthChanged`. Pseudocode:

```typescript
    private _bodyGraphics: Graphics;

    // In constructor: this._bodyGraphics = new Graphics(); this._container.addChild(this._bodyGraphics);

    private _drawBody(): void {
        this._bodyGraphics.clear();
        const bogies = this._engine.getBogies();
        if (bogies.length < 2) return;

        const dir = this._engine.lineDirection;
        const origin = this._engine.lineOrigin;
        const projections = bogies.map(b => {
            const dx = b.x - origin.x;
            const dy = b.y - origin.y;
            return dx * dir.x + dy * dir.y;
        });
        const min = Math.min(...projections) - 2.5; // edgeToBogie default
        const max = Math.max(...projections) + 2.5; // bogieToEdge default
        const halfW = this._engine.width / 2;
        const normal = { x: -dir.y, y: dir.x };

        // Compute four corners
        const p = (t: number, n: number) => ({
            x: origin.x + dir.x * t + normal.x * n,
            y: origin.y + dir.y * t + normal.y * n,
        });
        const c1 = p(min, halfW);
        const c2 = p(max, halfW);
        const c3 = p(max, -halfW);
        const c4 = p(min, -halfW);

        this._bodyGraphics.moveTo(c1.x, c1.y);
        this._bodyGraphics.lineTo(c2.x, c2.y);
        this._bodyGraphics.lineTo(c3.x, c3.y);
        this._bodyGraphics.lineTo(c4.x, c4.y);
        this._bodyGraphics.closePath();
        this._bodyGraphics.stroke({
            color: 0xaaaaaa,
            width: 1.0 / this._zoomLevel,
        });
    }
```

Subscribe to width changes in `setup()`:

```typescript
this._unsubscribeWidth = this._engine.onWidthChanged(() => this._drawBody());
```

Also call `_drawBody()` from `_syncAll()` and on bogie position change.

Clean up the listener in `cleanup()`.
